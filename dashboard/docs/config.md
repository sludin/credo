# Dashboard Configuration Reference

The Dashboard BFF (backend-for-frontend) loads its config from `dashboard.config.json` in the current directory, or from the path in `DASHBOARD_CONFIG_PATH`.

Config files support variable interpolation (`${VAR}`) from a `vars` block and from environment variables, and `includes` arrays for splitting config across multiple files. Keys prefixed with `_` are stripped before parsing (use them as JSON comments).

## Meta Fields

| Field | Type | Description |
|-------|------|-------------|
| `vars` | object | Variable definitions, referenced as `${name}` elsewhere in the config |
| `includes` | string[] | Paths to additional JSON config files to merge in (resolved relative to this file) |
| `baseDir` | string | Base directory for resolving relative paths. Defaults to the config file's directory |

## Network

| Field | Type | Default | Env override | Description |
|-------|------|---------|--------------|-------------|
| `port` | number | `7030` | `PORT` | Port for the BFF HTTPS server |
| `bind` | string | `"127.0.0.1"` | `BIND` | Interface to bind. The default restricts the BFF to loopback; set to `"0.0.0.0"` only when behind a trusted reverse proxy |

## Shepherd Connection

| Field | Type | Default | Env override | Description |
|-------|------|---------|--------------|-------------|
| `shepherdApiUrl` | string | **required** | `SHEPHERD_API_URL` | Base URL of Shepherd's dashboard API (e.g. `"https://shepherd.example.com:7011"`) |
| `caPath` | string | — | `DASHBOARD_CA_PATH` | CA bundle used to verify Shepherd's server certificate. Shared fallback for both `tls.clientCaPath` and `mtls.caPath` when those are not set individually |
| `requestTimeoutSeconds` | number | `15` | `DASHBOARD_REQUEST_TIMEOUT_SECONDS` | Timeout for outbound requests to Shepherd |

## TLS (inbound — browser connections)

The certificate the BFF presents to browsers. Both fields are required.

| Field | Type | Default | Env override | Description |
|-------|------|---------|--------------|-------------|
| `tls.certPath` | string | **required** | `DASHBOARD_TLS_CERT_PATH` | Server certificate (PEM, full chain) |
| `tls.keyPath` | string | **required** | `DASHBOARD_TLS_KEY_PATH` | Server private key (PEM) |

## mTLS (outbound — Shepherd connections)

The client certificate the BFF presents when calling Shepherd's dashboard API.

| Field | Type | Default | Env override | Description |
|-------|------|---------|--------------|-------------|
| `mtls.certPath` | string | **required** | `DASHBOARD_MTLS_CERT_PATH` | Client certificate (PEM, full chain) |
| `mtls.keyPath` | string | **required** | `DASHBOARD_MTLS_KEY_PATH` | Client private key (PEM) |
| `mtls.caPath` | string | falls back to `caPath` | `DASHBOARD_MTLS_CA_PATH` | CA bundle for verifying Shepherd's server certificate |
| `mtls.rejectUnauthorized` | bool | `true` | `DASHBOARD_MTLS_REJECT_UNAUTHORIZED` | Verify Shepherd's certificate. **Never set to `false` in production** — it disables mTLS verification entirely |

## DNS Checker

Controls the in-memory DNS TXT polling jobs used by the DNS checker tool. Jobs are ephemeral and reset on BFF restart.

| Field | Type | Default | Env override | Description |
|-------|------|---------|--------------|-------------|
| `dnsPollingIntervalSeconds` | number | `5` | `DASHBOARD_DNS_POLLING_INTERVAL_SECONDS` | How often each DNS polling job queries resolvers |
| `dnsJobTimeoutSeconds` | number | `600` | `DASHBOARD_DNS_JOB_TIMEOUT_SECONDS` | How long a DNS polling job runs before expiring |
| `dnsPublicResolvers` | array | `[]` | — | Public DNS resolvers to query in addition to authoritative nameservers. Each entry: `{ "name": "Google", "ip": "8.8.8.8" }` |

## Auth

All `auth.*` fields live under the `auth` key.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `auth.usersPath` | string | `"dashboard.users.json"` | Path to the user registry JSON file. Managed by `dashboard user` CLI commands |
| `auth.sessionSecret` | string | **required** | Secret for signing session cookies. Must be a strong random value; see [Operator Hardening](../../docs/operator-hardening.md) |
| `auth.sessionDurationHours` | number | `24` | How long a passkey session lasts before requiring re-authentication |
| `auth.enrollmentTokenTTLHours` | number | `24` | How long enrollment URLs generated by `dashboard user create` and `dashboard user reset` remain valid |
| `auth.rpId` | string | **required** | WebAuthn Relying Party ID — must match the hostname users access the dashboard on (e.g. `"dashboard.example.com"`) |
| `auth.rpName` | string | `"Credo Dashboard"` | Human-readable name shown to users in passkey prompts |
| `auth.origin` | string | `"https://<rpId>"` | Expected origin for WebAuthn assertions. Defaults to `https://<auth.rpId>` |
| `auth.identityEnvironment` | string | `"prod"` | Label embedded in enrollment tokens for validation |
| `auth.roleRefreshIntervalSeconds` | number | `300` | How often the BFF re-fetches a user's role from Shepherd |
| `auth.roleStaleTimeoutSeconds` | number | `1800` | How long a cached role is considered valid if a refresh fails |
| `auth.sessionsDir` | string | `"sessions"` | Directory for persisting session state across BFF restarts |

> **Legacy field names:** `auth.roleRefreshIntervalMs` and `auth.roleStaleTimeoutMs` (milliseconds) are accepted for backward compatibility and converted automatically. Use the `*Seconds` names in new configs.

## Example config

```json
{
  "port": 7030,
  "bind": "127.0.0.1",
  "shepherdApiUrl": "https://shepherd.example.com:7011",

  "caPath": "/etc/credo/credo-catrust.pem",

  "tls": {
    "certPath": "/var/apps/dashboard/certs/dashboard.example.com/fullchain.pem",
    "keyPath":  "/var/apps/dashboard/certs/dashboard.example.com/privkey.pem"
  },
  "mtls": {
    "certPath":           "/var/apps/dashboard/certs/dashboard.example.com/fullchain.pem",
    "keyPath":            "/var/apps/dashboard/certs/dashboard.example.com/privkey.pem",
    "caPath":             "/etc/credo/credo-catrust.pem",
    "rejectUnauthorized": true
  },

  "requestTimeoutSeconds": 15,
  "dnsPollingIntervalSeconds": 5,
  "dnsJobTimeoutSeconds": 600,
  "dnsPublicResolvers": [
    { "name": "Google",     "ip": "8.8.8.8" },
    { "name": "Cloudflare", "ip": "1.1.1.1" }
  ],

  "auth": {
    "usersPath":                 "/var/apps/dashboard/dashboard.users.json",
    "sessionSecret":             "output-of-openssl-rand-hex-32",
    "sessionDurationHours":      24,
    "enrollmentTokenTTLHours":   24,
    "rpId":                      "dashboard.example.com",
    "rpName":                    "Credo Dashboard",
    "origin":                    "https://dashboard.example.com",
    "identityEnvironment":       "prod",
    "roleRefreshIntervalSeconds": 300,
    "roleStaleTimeoutSeconds":   1800,
    "sessionsDir":               "/var/apps/dashboard/sessions"
  }
}
```

## `dashboard.users.json` schema

Managed by `dashboard user` CLI commands and the BFF enrollment flow. Do not edit directly while the BFF is running.

```json
{
  "users": [
    {
      "id":                     "uuid-v4",
      "shepherdAccount":        "alice-admin",
      "displayName":            "Alice Admin",
      "email":                  "alice@example.com",
      "active":                 true,
      "identityUri":            "vigil://credo/prod/admin/alice",
      "passkeys":               [],
      "pendingInvite":          null,
      "shepherdAccessToken":    "...",
      "shepherdRefreshToken":   "...",
      "shepherdTokenExpiresAt": "2026-01-15T13:00:00.000Z"
    }
  ]
}
```

| Field | Description |
|-------|-------------|
| `id` | UUID assigned at creation |
| `shepherdAccount` | Shepherd account name this user is linked to |
| `displayName` | Display name shown in the UI |
| `email` | User's email address |
| `active` | Inactive users cannot log in |
| `identityUri` | Vigil URI SAN; must match the user's Vigil certificate |
| `passkeys` | Registered WebAuthn credentials (populated by the enrollment flow) |
| `pendingInvite` | Hashed enrollment token + expiry; `null` when enrolled |
| `shepherdAccessToken` | JWT access token from Shepherd, used for proxied API calls |
| `shepherdRefreshToken` | Refresh token for obtaining new access tokens |
| `shepherdTokenExpiresAt` | ISO timestamp of access token expiry |
